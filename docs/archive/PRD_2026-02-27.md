# Product Requirements Document (PRD)

## 1. Executive Summary
This project aims to enable **Statamic** sites to participate in the **Fediverse** (ActivityPub network). It consists of a suite of **modular addons** that provide core federation capabilities and specialized content types (Articles, Questions/Polls).

## 2. Strategic Context
### Vision
To make Statamic a first-class citizen in the decentralized web, allowing content creators to own their distribution while connecting with millions of users on Mastodon, Pixelfed, and other ActivityPub platforms.

### Goals
- **Seamless Integration**: Native Statamic experience (Control Panel, Antlers).
- **Modularity**: Core package for protocols, separate packages for content types.
- **Performance**: Capable of handling high-volume federation (thousands of followers).
- **Reliability**: Robust error handling, retries, and dead letter queues.

## 3. Architecture
### Addon Structure
- **ActivityPubCore**: The "engine". Handles actors, inboxes, outboxes, HTTP signatures, and job dispatching.
- **ActivityPubLongFormat**: Adds `Article` type support (blogs, essays).
- **ActivityPubQuestions**: Adds `Question` type support (polls).

### Key Components
- **Data Storage**: Flat-file (content) + SQLite (queue/performance data).
- **Queue System**: Laravel Database Queue (optimized for batching and rate limiting).
- **Security**: HTTP Signatures (RSA-SHA256) + strict content verification.

## 5. Technical Constraints & Standards
- **Framework**: Statamic 5.x / 6.x / Laravel 12.x
- **PHP**: 8.2+ (Strict Types enforced)
- **Testing**: PHPUnit 11.x (High coverage required)
- **Code Style**: PSR-12, tough linting.

## 6. Addon Expandability
To maintain a clean and modular architecture, the core package must provide foundational hooks and extension points for other targeted functionality.
- **Modular Vue Components**: The Control Panel UI must support dynamic component registration and extensibility.
  - Addons must be able to specify custom, individual formatting for the "All" Inbox stream presentation based on generic or unknown object types.
  - Addons must have the ability to extend the primary "New" button (e.g., adding "New Note", "New Check-in", "New Poll", etc.).
  - Addons must have the ability to add new Stacks and Drawers for specialized processing and full-screen presentation.
